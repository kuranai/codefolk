# Changelog

All notable changes to Codefolk are documented here.

## [Unreleased]

## [0.1.3] - 2026-07-22

### Changed

- Removed dashed hover outlines from toolbar actions and navigation tabs in both themes.

## [0.1.2] - 2026-07-22

### Changed

- Removed the global dashed focus outline from interactive workbench controls in both themes.

## [0.1.1] - 2026-07-22

### Changed

- Restored escook Dark's characteristic editor palette, subdued workbench chrome, differentiated tabs, and classic Git status colors while retaining modern Codefolk workbench coverage.
- Improved the Dark Activity Bar's active and hover states for faster orientation and reliable icon contrast.

## [0.1.0] - 2026-07-22

### Added

- Codefolk Dark and Codefolk Light themes derived from the escook Dark and Light palettes.
- Curated coverage for 553 VS Code 1.129.1 workbench color IDs per theme.
- Semantic highlighting for declarations, types, functions, parameters, properties, readonly values, and deprecated symbols.
- Modern editor, terminal, notebook, SCM/merge, debug, testing, chat, and workbench colors.
- Deterministic TypeScript theme generator, registry validation, contrast checks, visual fixtures, VSIX packaging, and release automation.

### Changed

- Restored escook Light's characteristic editor palette for strings, keywords, constants, functions, parameters, types, and comments while retaining modern Codefolk workbench coverage.
- Restored escook Light's stronger active Activity Bar item, quieter SCM section headers and inactive tabs, and classic VS Code Light Git status colors.
